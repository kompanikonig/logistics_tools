# Логистика 2026 — панель загрузки и стендап

Набор внутренних инструментов планирования (статичные одностраничные HTML, без сборки).
Хостинг — **GitHub Pages**, автодеплой по `git push` в ветку `main`.

## Живые адреса

- **Дашборд загрузки:** https://kompanikonig.github.io/logistics_tools/
- **Стендап-доска:** https://kompanikonig.github.io/logistics_tools/standup.html

## Состав

| Файл | Что это |
|------|---------|
| `index.html` | **Панель загрузки и сроков** — тепловая карта загрузки по месяцам (с разбивкой по проектам при клике на ячейку), детализация по проектам и исполнителям, блок «Ресурсы к планированию», мультифильтры по руководителю и компетенции, дрилл-дауны, ссылки в Jira/CAB. Данные читаются из Google Sheets (CSV-публикация). |
| `standup.html` | **Стендап-доска команды** — ежедневный план в свободной форме, статусы задач (в работе / выполнено / не сделано / перенесено), статус человека на день (работаю / отпуск / болею), архив по дням, перенос невыполненных. Прототип: данные хранятся во встроенном хранилище страницы. |
| `apps-script/SmartsheetSync.gs` | Google Apps Script: односторонняя синхронизация Google Sheets в Smartsheet (кнопка «Smartsheet» в меню таблицы). В репозитории лежит как исходник/бэкап — ставится в саму Google-таблицу. |

## Источники данных

- **Дашборд** читает опубликованный CSV из Google Sheets (лист «Логистика 2026»). Колонки определяются по именам, а не по позициям — добавление или перестановка колонок дашборд не ломает.
- **Синк** Google в Smartsheet запускается кнопкой «Smartsheet» в самой Google-таблице.
- CSV-ссылка зашита в `index.html` (абсолютная), поэтому данные грузятся независимо от адреса сайта.

## Рабочий процесс (правка и деплой)

```bash
# внёс правки в index.html или standup.html
git add .
git commit -m "что изменил"
git push            # GitHub Pages пересоберёт сайт автоматически (~1-2 мин)
```

> Перетаскивать файлы куда-либо не нужно — деплой происходит из ветки `main` сам.

## Локальный просмотр

```bash
# из корня репозитория
python3 -m http.server 8000
# дашборд:  http://localhost:8000
# стендап:  http://localhost:8000/standup.html
```

> Открывать через `http://localhost`, а не двойным кликом по файлу: `file://` блокирует загрузку CSV из-за CORS.

## Как настроен хостинг (справочно)

GitHub Pages включён в настройках репозитория: Settings -> Pages -> Build and deployment
- Source: **Deploy from a branch**
- Branch: **main**, папка **/ (root)**

Любой коммит в `main` публикуется автоматически.

## Заметки

- Репозиторий публичный — сайт доступен по ссылке без пароля (внутренние данные планирования видны всем, у кого есть адрес).
- Дата в логике дашборда и стендапа берётся из окружения (текущая).
- Файлы самодостаточны: весь CSS/JS внутри каждого HTML, внешних зависимостей нет.
- В планах: перевод стендап-доски на Google Sheets как базу (двусторонняя связь для общего доступа команды) и объединение дашборда со стендапом во вкладки.
