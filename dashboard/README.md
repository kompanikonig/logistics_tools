## Живые адреса

- **Дашборд загрузки:** https://kompanikonig.github.io/logistics_tools/

## Состав репозитория

| Файл | Что это |
|------|---------|
| `index.html` | Панель загрузки и сроков (основной дашборд). |

## Что умеет дашборд

**Данные**
- Живая загрузка из Google Sheets (опубликованный CSV листа «Логистика 2026»).
- Колонки определяются по именам, а не по позициям — добавление/перестановка/переименование колонок дашборд не ломает (например, «Исполнитель» переименован в «Ресурс» — подхватывается).
- Кнопка «Обновить данные» и индикатор статуса: онлайн / оффлайн-снимок, число строк, время обновления.
- Встроенный снимок данных как резерв на случай оффлайна.

**Вкладка «Обзор»**
- KPI-карточки (проекты в выборке, занятые исполнители, без назначения) — за весь год.
- Тепловая карта загрузки: исполнитель × 12 месяцев, цвет = уровень загрузки, плюс среднегодовая колонка.
- Загрузка считается из «Выделено %» и периода задачи (помесячно).
- Легенда-фильтр по зонам загрузки (свободен / оптимум / перегруз / не занят) с мультивыбором.
- **Клик по ячейке тепловой карты** — модалка с разбивкой: из каких проектов сложилась загрузка в этом месяце, с процентами и ссылками в Jira/CAB.
- Блок **«Ресурсы к планированию»**:
  - «Свободные ресурсы» — у кого есть запас по месяцам (% до полной ставки), цвет = размер запаса.
  - «Нужен ресурс» — проекты без назначенного исполнителя (требуемая загрузка по месяцам).
  - Фильтр по компетенциям (SAP EWM / SAP_LO / РП), мультивыбор.
  - Переключатель горизонта: «с текущего месяца» (по умолчанию) / «весь год».

**Вкладка «Детализация»**
- Группировка «По проектам» и «По исполнителям».
- Одна строка на сущность: в «По исполнителям» — помесячная суммарная загрузка человека; в «По проектам» — участники схлопнуты в одну строку каждый, с помесячной загрузкой на проекте.
- Цвет полос загрузки = уровень загрузки; шапка проекта — со сроком, дедлайном и статусом.
- Сортировка (по старту / дедлайну / загрузке / имени).
- Клик по строке — дрилл-даун с деталями (период, дни до дедлайна, руководитель, компетенции, разбивка).
- **Кликабельные проекты → Jira/CAB**: бейдж со ссылкой на тикет (из колонки «Номер CAB»), открывается в новой вкладке.

**Вкладка «Загрузка БА»**
- Отдельная помесячная карта загрузки бизнес-аналитиков.

**Общее**
- Мультифильтры (чекбоксы) по руководителю и компетенции + текстовый поиск.
- Тёмная / светлая тема.
- Приглушённая палитра руководителей, единые цвета загрузки.

## Синхронизация в Smartsheet (`apps-script/SmartsheetSync.gs`)

- Односторонний полный зеркальный синк Google Sheets → лист-приёмник в Smartsheet (запуск кнопкой в меню таблицы).
- Сопоставление колонок по именам; обработка дат, чисел, чекбокса «В реестре 2025», дропдауна компетенций.
- Новые/изменённые/удалённые строки — все переносятся; «лишние» колонки приёмника (общее планирование) не затрагиваются.

## Бэклог и планы

- **Кросс-направленческая загрузка**: подтянуть данные соседних направлений из общего Smartsheet и видеть полную занятость ресурса по всем направлениям.
- **Стендап на Google Sheets как базе**: перевод с прототипа на общее хранилище (двусторонняя связь через Apps Script Web App), чтобы команда писала с разных устройств.
- **Объединение** дашборда и стендапа во вкладки одного приложения.
- Статусы проектов и «следующий шаг», блок «ближайшие дедлайны», экспорт в PDF.
- Запоминание темы и фильтров между визитами.
